## Topic: $ARGUMENTS

Based on the topic argument, select the appropriate profile file:
- `relationships` → `07 Context/relationships.md`
- `career` or `work` → `07 Context/career-projects.md`
- `health` or `habits` → `07 Context/health-habits.md`
- `core`, `values`, `identity` → `07 Context/core-profile.md`
- No argument or `random` → Pick the file with most gaps/oldest `last_updated`

## Steps

1. Read the selected profile file
2. Identify ONE area where information is missing, outdated, or could be deeper (do this silently, user don't need to know what you pick and why)
3. Ask me a clear, conversational question about it
   - Make it specific, not generic
   - Reference what you already know to show context
   - Examples: "You mentioned X - how has that evolved?" or "I don't have much about Y - can you tell me more?"
4. After I answer:
   - Update the appropriate profile file with new information
   - Update `last_updated` in frontmatter
   - Keep formatting consistent with existing structure

Reply only with your question (don't explain what files you read, user already know that). After I answer, reply only: "✓ Profile updated"
